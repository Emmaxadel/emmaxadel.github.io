Project Overview

This project presents a comprehensive sentiment analysis of malaria-related discussions on Twitter across Nigeria. By scraping tweets from major Nigerian news outlets and applying natural language processing techniques, the analysis identifies malaria hotspots and public sentiment patterns across 16 Nigerian states. The project combines web scraping, NLP, sentiment analysis, and geospatial visualization to provide actionable insights for public health decision-making.

Key Objectives

  • Scrape and analyze malaria-related tweets from major Nigerian news sources
  • Extract location data and perform named entity recognition (NER) on tweet content
  • Conduct sentiment analysis using VADER to classify public sentiment
  • Create interactive geospatial visualizations showing sentiment distribution across Nigeria
  • Identify states requiring urgent public health attention based on negative sentiment
  • Provide data-driven recommendations for malaria prevention and intervention programs

Methodology & Workflow

1. Data Collection

Utilized the Twikit library to scrape tweets containing keywords "malaria," "mosquito," or "mosquitoes" from major Nigerian news outlets (Punch Newspapers, The Nation, Guardian Nigeria, Nigerian Tribune, Vanguard, Daily Post, The Sun Nigeria) from September 2023 onwards. Collected over 5,000 tweets with comprehensive metadata including timestamps, likes, retweets, and location data.

2. Named Entity Recognition & Location Extraction

Applied NLTK's natural language processing capabilities to extract location entities from tweet text. Implemented custom filtering to remove irrelevant terms and mapped cities to their parent Nigerian states. Normalized location names and handled variations (e.g., "Lagos State" ‚Üí "LAGOS").

3. Sentiment Analysis

Employed VADER (Valence Aware Dictionary and sEntiment Reasoner) sentiment analyzer to classify tweets as positive (compound score ‚â• 0.05), negative (‚â§ -0.05), or neutral. Preprocessed text by removing URLs, mentions, hashtags, and special characters to improve analysis accuracy.

4. Geospatial Visualization

Created interactive maps using Folium library, centered on Nigeria (9.0820¬∞N, 8.6753¬∞E). Aggregated sentiment scores at state level and visualized using color-coded markers (green = positive, orange = neutral, red = negative) with custom popups displaying sentiment intensity, mention counts, and detailed statistics.

# Nigeria Malaria Sentiment Analysis - Python Code
# Twitter Data Scraping, NLP, and Geospatial Visualization

import nltk
from nltk import ne_chunk, pos_tag, word_tokenize
from nltk.tree import Tree

import folium
from folium.plugins import MarkerCluster

# ============================================
# PART 1: DATA COLLECTION (Twitter Scraping)
# ============================================

from twikit import Client, TooManyRequests
import time
from datetime import datetime
import csv
from configparser import ConfigParser

# Query configuration
QUERY = '(malaria OR mosquito OR mosquitoes) (from:MobilePunch OR from:TheNationNews OR from:GuardianNigeria OR from:nigeriantribune OR from:vanguardngrnews OR from:DailyPostNGR OR from:thesunnigeria) since:2023-09-01'

# Authenticate to Twitter
client = Client(language='en-US')

print("Scraping malaria-related tweets from Nigerian news sources...")

# ============================================
# PART 2: LOAD AND EXPLORE DATA
# ============================================

# Load scraped Twitter data
df = pd.read_csv("twitter_data.csv")
print(f"Dataset loaded: {df.shape[0]} tweets")
print("\nFirst few rows:")
print(df.head())

# Check for missing values
print("\nMissing values:")
print(df.isnull().sum())

# ============================================
# PART 3: LOCATION EXTRACTION (NER)
# ============================================

print("\n=== Extracting Location Entities ===")

# Function to extract locations using NLTK's NER
def extract_locations(text):
    try:
        tokens = word_tokenize(text)
        pos_tags = pos_tag(tokens)
        chunks = ne_chunk(pos_tags, binary=False)
        
        locations = []
        for chunk in chunks:
            if isinstance(chunk, Tree) and chunk.label() == 'GPE':
                locations.append(' '.join([token for token, pos in chunk.leaves()]))
        return locations
    except:
        return []

# Apply location extraction
df['extracted_locations'] = df['Text'].apply(extract_locations)

# List of irrelevant terms to filter out
irrelevant_terms = {
    "malaria", "mosquitoes", "mosquito", "untreatable", "n20bn", 
    "burna", "yhemolee", "nddc", "otedola", "ui", "us", 
    "east africa", "kenya"
}

# Function to clean extracted locations
def clean_locations(locations):
    clean_locs = []
    for loc in locations:
        # Ignore irrelevant terms
        if loc.lower() in irrelevant_terms:
            continue
        
        # Ignore URLs
        if re.match(r"http[s]?://", loc):
            continue
        
        # Ignore single characters
        if len(loc) <= 1:
            continue
        
        # Add cleaned location
        clean_locs.append(loc)
    
    return clean_locs if clean_locs else None

# Clean the extracted locations
df['cleaned_locations'] = df['extracted_locations'].apply(clean_locations)

# Handle location variations and normalize
df['cleaned_locations'] = df['cleaned_locations'].astype(str)
df['cleaned_locations'].replace({
    "Lagos, Nigeria": "['Lagos']", 
    "['Lagos, Nigeria']": "['Lagos']"
}, inplace=True)

# Convert to uppercase for consistency
df['cleaned_locations'] = df['cleaned_locations'].str.upper()

print("\nLocation extraction complete!")
print(f"Found locations in {df['cleaned_locations'].notna().sum()} tweets")

# ============================================
# PART 4: TEXT CLEANING
# ============================================

print("\n=== Cleaning Tweet Text ===")

# Function to clean tweet text
def clean_text(text):
    text = re.sub(r'http\S+', '', text)  # Remove URLs
    text = re.sub(r'@\w+', '', text)  # Remove mentions
    text = re.sub(r'#', '', text)  # Remove hashtags
    text = re.sub(r'[^A-Za-z\s]', '', text)  # Remove special characters
    return text

# Clean the text column
df['cleaned_text'] = df['Text'].apply(clean_text)

print("Text cleaning complete!")

# ============================================
# PART 5: SENTIMENT ANALYSIS (VADER)
# ============================================

print("\n=== Performing Sentiment Analysis ===")

# Initialize VADER sentiment analyzer
analyzer = SentimentIntensityAnalyzer()

# Function to classify sentiment using VADER
def classify_vader_sentiment(text):
    score = analyzer.polarity_scores(text)
    if score['compound'] >= 0.05:
        return 'good'
    elif score['compound'] <= -0.05:
        return 'bad'
    else:
        return 'neutral'

# Apply sentiment analysis
df['sentiment'] = df['cleaned_text'].apply(classify_vader_sentiment)

print("\nSentiment Analysis Results:")
print(df['sentiment'].value_counts())
print("\nSample results:")
print(df[['cleaned_text', 'sentiment']].head())

# ============================================
# PART 6: GEOCODING & COORDINATE MAPPING
# ============================================

print("\n=== Mapping Locations to Coordinates ===")

# Nigerian state coordinates dictionary
nigerian_coordinates = {
    'LAGOS': (6.5244, 3.3792),
    'JIGAWA': (12.2289, 9.5616),
    'TARABA': (7.9999, 10.7739),
    'BAUCHI': (10.3158, 9.8442),
    'DELTA': (5.8903, 5.6803),
    'OYO': (7.8451, 3.9470),
    'EKITI': (7.7190, 5.3110),
    'BORNO': (11.8333, 13.1500),
    'ANAMBRA': (6.2209, 6.9370),
    'ABIA': (5.4527, 7.5248),
    'EDO': (6.3350, 5.6220),
    'KWARA': (8.9670, 4.3840),
    'NASARAWA': (8.5372, 8.3150),
    'ONDO': (7.2500, 5.1950),
    'ZAMFARA': (12.1704, 6.6594),
    'FCT ABUJA': (9.0765, 7.3986),
}

# Map cities to their parent states
city_to_state_mapping = {
    'MAKOKO': 'LAGOS',
    'IBADAN': 'OYO',
    'BENIN': 'EDO',
    'BENIN CITY': 'EDO',
    'ONITSHA': 'ANAMBRA',
    'ONITSHA, NIGERIA': 'ANAMBRA',
    'AKURE': 'ONDO',
    'PORT HARCOURT': 'RIVERS',
    'KANO CITY': 'KANO',
    'MAIDUGURI': 'BORNO',
    'ABUJA': 'FCT ABUJA',
    'OLUYOLE': 'OYO',
    'SAUCHI': 'BAUCHI',
}

# Function to get coordinates for a location
def get_coordinates(location_str):
    if pd.isna(location_str) or location_str == 'None':
        return (None, None)
    
    # Extract location from string representation
    location = eval(location_str)[0] if location_str.startswith('[') else location_str
    location_upper = location.upper()
    
    # Map city to state if needed
    if location_upper in city_to_state_mapping:
        location_upper = city_to_state_mapping[location_upper]
    
    # Return coordinates if found
    if location_upper in nigerian_coordinates:
        return nigerian_coordinates[location_upper]
    
    return (None, None)

# Apply coordinate mapping
df['lat_long'] = df['cleaned_locations'].apply(get_coordinates)
df['latitude'] = df['lat_long'].apply(lambda x: x[0])
df['longitude'] = df['lat_long'].apply(lambda x: x[1])

# Remove rows without coordinates
df = df.dropna(subset=['latitude', 'longitude'])

print(f"Successfully mapped {len(df)} tweets to coordinates")

# ============================================
# PART 7: STATE-LEVEL AGGREGATION
# ============================================

print("\n=== Aggregating Sentiment by State ===")

# Convert sentiment to numerical values
sentiment_values = {'good': 1, 'neutral': 0, 'bad': -1}
df['sentiment_value'] = df['sentiment'].map(sentiment_values)

# Group by state and aggregate
state_sentiment = df.groupby('cleaned_locations').agg({
    'sentiment': 'count',  # Count of mentions
    'sentiment_value': 'mean',  # Average sentiment
    'latitude': 'first',
    'longitude': 'first'
}).reset_index()

# Convert average sentiment back to categories
def sentiment_from_value(value):
    if value > 0:
        return 'good'
    elif value < 0:
        return 'bad'
    else:
        return 'neutral'

state_sentiment['average_sentiment'] = state_sentiment['sentiment_value'].apply(sentiment_from_value)
state_sentiment.rename(columns={'sentiment': 'mention_count'}, inplace=True)

print("\nState-level sentiment summary:")
print(state_sentiment[['cleaned_locations', 'mention_count', 'average_sentiment']].head(10))

# ============================================
# PART 8: INTERACTIVE MAP VISUALIZATION
# ============================================

print("\n=== Creating Interactive Sentiment Map ===")

# Create base map centered on Nigeria
m = folium.Map(location=[9.0820, 8.6753], zoom_start=6, tiles='OpenStreetMap')

# Define color mapping for sentiment
sentiment_colors = {
    'good': '#27ae60',    # Green
    'neutral': '#f39c12',  # Orange
    'bad': '#e74c3c'       # Red
}

sentiment_icons = {
    'good': 'üòä',
    'neutral': 'üòê',
    'bad': 'üòü'
}

# Add markers for each state
for idx, row in state_sentiment.iterrows():
    sentiment = row['average_sentiment']
    color = sentiment_colors[sentiment]
    icon = sentiment_icons[sentiment]
    
    # Create popup content
    popup_html = f"""
    

        

            üìç {row['cleaned_locations']}
        

        

        

            Overall Sentiment: {sentiment.upper()} {icon}

            Total Mentions: {row['mention_count']}

            Sentiment Intensity: {abs(row['sentiment_value']):.2f}
        

    

    """
    
    # Add marker to map
    folium.Marker(
        location=[row['latitude'], row['longitude']],
        popup=folium.Popup(popup_html, max_width=300),
        tooltip=f"{row['cleaned_locations']}: {sentiment.upper()}",
        icon=folium.Icon(color='red' if sentiment == 'bad' else 
                        'orange' if sentiment == 'neutral' else 'green',
                        icon='info-sign')
    ).add_to(m)

# Save the map
m.save('malaria_sentiment_map.html')
print("\nInteractive map saved as 'malaria_sentiment_map.html'")

# ============================================

positive_states = len(state_sentiment[state_sentiment['average_sentiment'] == 'good'])
neutral_states = len(state_sentiment[state_sentiment['average_sentiment'] == 'neutral'])
negative_states = len(state_sentiment[state_sentiment['average_sentiment'] == 'bad'])

most_discussed = state_sentiment.loc[state_sentiment['mention_count'].idxmax(), 'cleaned_locations']
most_discussed_count = state_sentiment['mention_count'].max()

# States needing attention (negative sentiment)
attention_states = state_sentiment[state_sentiment['average_sentiment'] == 'bad']['cleaned_locations'].tolist()

print(f"\nüìä OVERVIEW:")

Technical Challenges & Solutions

Data Collection Rate Limits

Challenge: Twitter API rate limits restricted data collection speed.

Solution: Implemented intelligent wait mechanisms and retry logic to handle TooManyRequests exceptions, ensuring continuous data collection without API violations.

Location Ambiguity

Challenge: Many tweets lacked explicit location information or contained ambiguous references.

Solution: Combined profile location data with NER-extracted locations, created comprehensive city-to-state mappings, and filtered irrelevant terms to improve accuracy.

Sentiment Context Sensitivity

Challenge: Medical discussions can be contextually complex, requiring nuanced sentiment interpretation.

Solution: Used VADER's compound score with carefully calibrated thresholds (¬±0.05) to distinguish between genuinely positive/negative content and neutral medical reporting.

State-Level Aggregation

Challenge: Aggregating individual tweets to state-level insights while preserving sentiment nuances.

Solution: Implemented weighted averaging of sentiment scores, tracking both mention counts and sentiment intensity to provide comprehensive state-level assessments.
